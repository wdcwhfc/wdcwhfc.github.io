Java 序列化与反序列化

Sep 04, 2025
3 minute read

0x00 基本概念

  • 序列化：将Java对象转换成字节流的过程。通过java.io.ObjectOutputStream类的writeObject()方法实现。

  • 反序列化：序列化的反过程，可以将字节流还原为Java对象。通过java.io.ObjectInputStream类的readObject()方法实现。

  • 序列化对象必须实现java.io.Serializable(内部序列化)或java.io.Externalizable(外部序列化)接口。Serializable接口没有什么实际内容，基本只是作为一个类是否能序列化的标识使用。

  • 实现了Serializable接口的类，其所有属性都可以被序列化其才可以被序列化或反序列化。（用transient关键字修饰的属性除外，不参与序列化过程）。

  • 反序列化操作时，必须满足以下两个条件：

    • 被反序列化的类必须存在
    • serialVersionUID必须相同

  • 反序列化生成类实例时，不会调用该类的构造方法。

主体类

 1import java.io.Serializable;
 2
 3public class User implements Serializable { 
 4    private static final long serialVersionUID = 1L;
 5    private String name;
 6    private int age;
 7    
 8    public User() {
 9        System.out.println("User 默认构造方法被调用。");
10    }
11
12    public User(String name, int age) {
13        this.name = name;
14        this.age = age;
15        System.out.println("User 构造方法被调用。");
16    }
17
18    public String getName() {
19        return name;
20    }
21
22    public int getAge() {
23        return age;
24    }
25}

序列化与反序列化

 1import java.io.ByteArrayOutputStream;
 2import java.io.ObjectOutputStream;
 3import java.io.ByteArrayInputStream;
 4import java.io.ObjectInputStream;
 5
 6public class Main {
 7    public static void main(String[] args)
 8    {
 9        User user = new User("b5juan", 18);
10
11        // 序列化
12        ByteArrayOutputStream outputStream = new ByteArrayOutputStream();
13        try (ObjectOutputStream oos = new ObjectOutputStream(outputStream)){
14            oos.writeObject(user);
15        }
16        catch (Exception e) {
17            e.printStackTrace();
18        }
19
20        byte[] bytes = outputStream.toByteArray();
21        System.out.println(getHexString(bytes));
22
23        // 反序列化
24        ByteArrayInputStream inputStream = new ByteArrayInputStream(bytes);
25        try (ObjectInputStream ois = new ObjectInputStream(inputStream)){
26            User user1 = (User) ois.readObject();
27            System.out.println("Name: " + user1.getName());
28            System.out.println(" Age: " + user1.getAge());
29        } catch (Exception e) {
30            throw new RuntimeException(e);
31        }
32    }
33
34    private static String getHexString(byte[] bytes)
35    {
36        StringBuilder hex = new StringBuilder();
37        for (byte b : bytes) {
38            hex.append(String.format("\\x%02x", b));
39        }
40        return hex.toString();
41    }
42}

0x02 自定义序列化

实现了Serializable接口的类，可以通过声明以下方法实现自定义序列化与反序列化过程：

  • private void writeObject(ObjectOutputStream oos): 自定义序列化过程。
  • private void readObject(ObjectInputStream ois): 自定义反序列化过程。
  • private void readObjectNoData(): 反序列化时，如果输入流为空，则调用此方法。用于版本不兼容等场景，可以输出一个兜底的对象。
  • protected void writeReplace(): 序列化前，调用此方法替换被序列化的对象。用于保护敏感信息、性能优化等场景，在 writeObject() 执行前调用。
  • protected Object readResolve(): 反序列化后，调用此方法替换反序列化生成的对象，返回值为最终的对象。在 readObject() 执行之后调用。

writeObject、readObject、readObjectNoData必须由private修饰，调用时JVM通过反射方式调用。writeReplace、readResolve 建议为 protected，允许子类覆盖。

示例

 1import java.io.Serializable;
 2import java.io.ObjectOutputStream;
 3import java.io.ObjectInputStream;
 4
 5public class User implements Serializable { 
 6    private static final long serialVersionUID = 1L;
 7    private String name;
 8    private int age;
 9    
10    // ...
11
12    public User(String name, int age) {
13        this.name = name;
14        this.age = age;
15        System.out.println("User 构造方法被调用。");
16    }
17
18    // ...
19
20
21    private void writeObject(ObjectOutputStream oos) throws Exception {
22        System.out.println("writeObject 执行了");
23        oos.defaultWriteObject();
24    }
25
26    private void readObject(ObjectInputStream ois) throws Exception {
27        System.out.println("readObject 执行了");
28        ois.defaultReadObject();
29    }
30
31    private void readObjectNoData() throws Exception {
32        System.out.println("readObjectNoData 执行了");
33    }
34
35    protected Object writeReplace() throws Exception {
36        System.out.println("writeReplace 执行了");
37        return new User("writeReplace", 1);
38    }
39
40    protected Object readResolve() throws Exception{
41        System.out.println("readResolve 执行了");
42        return new User("readResolve", 1);
43    }
44}
