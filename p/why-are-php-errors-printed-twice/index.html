Why are PHP errors printed twice?

Jul 28, 2023
1 minute read

问题

PHP cli 产生报错时，总是在终端打印两遍，且第一遍以“PHP”开头，第二遍直接以错误类型开头，譬如“PHP Parse error:”和“Parse error:”。

原因

第一遍以“PHP”开头的是php.ini中配置项“log_errors”的输出结果，当开启了该配置但没有指定 log 存放的位置时，该结果就会输出到错误输出（STDERR）中；

第二遍直接以错误类型开头的是配置项“display_errors”的输出结果，该结果输出到标准输出（STDOUT）中。

解决办法

1 、关闭一项配置

任意关闭上述两项配置中的一项即可解决，建议采用如下配置：

1
2

log_errors = On
display_errors = Off

1
2

ini_set('log_errors', 1);
ini_set('display_errors', 0);

2 、指定错误日志保存位置

在 php.ini 中指定 error_log的保存位置，“ log_errors”就不会输出到错误输出中，转而保存在指定的硬盘位置上。譬如：

1
2
3
4
5
6
7
8
9

; Log errors to specified file. PHP's default behavior is to leave this value
; empty.
; http://php.net/error-log
; Example:
;error_log = php_errors.log
; Log errors to syslog (Event Log on Windows).
;error_log = syslog

error_log = '/var/logs/php_error.log'

One more thing

“display_errors”配置开启时，输出结果默认输出到标准输出（STDOUT）中，但是也可以通过配置，使其输出到错误输出（STDERR）中。

1
2
3
4
5
6
7
8
9

; Possible Values:
;   Off = Do not display any errors
;   stderr = Display errors to STDERR (affects only CGI/CLI binaries!)
;   On or stdout = Display errors to STDOUT
; Default Value: On
; Development Value: On
; Production Value: Off

display_errors = On
