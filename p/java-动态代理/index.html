Java 动态代理

Aug 21, 2025
3 minute read

代理在Java中有着广泛的应用，比如Spring AOP、Hibernate数据查询、测试框架的后端mock、RPC远程调用、Java注解对象获取、日志、用户鉴权、全局性异常处理、性能监控，甚至事务处理等。

基础逻辑块

1public interface Service {
2    public void func_1();
3    public void func_2();
4}

 1public class ServiceImpl implements Service {
 2    @Override
 3    public void func_1()
 4    {
 5        System.out.println("ServiceImpl.func_1");
 6    }
 7
 8    @Override
 9    public void func_2()
10    {
11        System.out.println("ServiceImpl.func_2");
12    }
13}

静态代理

 1public class StaticServiceProxy implements Service {
 2    private Service service;
 3
 4    public StaticServiceProxy(Service service){
 5        this.service = service;
 6    }
 7
 8    @Override
 9    public void func_1(){
10        pre();
11        service.func_1();
12        post();
13    }
14
15    @Override
16    public void func_2(){
17        pre();
18        service.func_2();
19        post();
20    }
21
22    public void pre(){
23        System.out.println("pre of static proxy");
24    }
25
26    public void post(){
27        System.out.println("post of static proxy");
28    }
29}

 1public class ProxyMain {
 2    public static void main(String[] args) {
 3        Service service = new ServiceImpl();
 4        Service serviceStaticProxy = new StaticServiceProxy(service);
 5        serviceStaticProxy.func_1();
 6    }
 7}
 8
 9// 输出
10// pre of static proxy
11// ServiceImpl.func_1
12// post of static proxy

静态代理不够灵活，需要跟目标类实现同一接口。接口发生变化时，需要手动同步更改目标类和代理类。

当需要代理多个类时，不论是采取一个代理类对应一个业务类的方式还是一个代理类实现所有业务类接口的方式，都会使得代理类的代码很臃肿。

动态代理

JDK 动态代理

 1import java.lang.reflect.InvocationHandler;
 2import java.lang.reflect.Method;
 3
 4import com.b5juan.ProxyL.Service;
 5
 6public class JDKProxy implements InvocationHandler {
 7    private Service service;
 8
 9    public JDKProxy(Service service)
10    {
11        this.service = service;
12    }
13
14    // 被代理的类调用任意方法都会转到所绑定handler的invoke方法。
15    @Override
16    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
17        pre();
18        Object result = method.invoke(service, args);
19        post();
20        return result;
21    }
22
23    public void pre()
24    {
25        System.out.println("pre of JDK proxy");
26    }
27
28    public void post()
29    {
30        System.out.println("post of JDK proxy");
31    }
32    
33}

 1public class ProxyMain {
 2    public static void main(String[] args) {
 3        Service service = new ServiceImpl();
 4        // JDK 动态代理，通过反射实现。需要目标类实现了接口。
 5        InvocationHandler handler = new JDKProxy(service);
 6        // 传入业务类classloader、interfaces和实现了InvocationHandler接口的自定义handler对象
 7        // 获取与业务类实现相同接口的Proxy实例
 8        Service serviceDynamicProxy_1 = (Service) Proxy.newProxyInstance(service.getClass().getClassLoader(), service.getClass().getInterfaces(), handler);
 9        serviceDynamicProxy_1.func_2();
10    }
11}
12
13
14// 输出
15// pre of JDK proxy
16// ServiceImpl.func_2
17// post of JDK proxy

使用反射的方式实现代理，需要目标业务类实现了接口。

实现与目标业务类相同的接口，并把代理类的所有方法指向绑定的handler对象的 invoke() 方法。

CGLIB 动态代理

 1import net.sf.cglib.proxy.MethodInterceptor;
 2import net.sf.cglib.proxy.MethodProxy;
 3import java.lang.reflect.Method;
 4
 5public class CglibProxyInterceptor implements MethodInterceptor {
 6    // 被代理的类调用任意方法都会转到所绑定 interceptor 的 intercept 方法。
 7    @Override
 8    public Object intercept(Object object, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
 9        System.out.println("pre of CGLIB proxy");
10        Object result = methodProxy.invokeSuper(object, args);
11        System.out.println("post of CGLIB proxy");
12        return result;
13    }
14}

 1public class ProxyMain {
 2    public static void main(String[] args) {
 3        Service service = new ServiceImpl();
 4
 5        // CGLIB 动态代理，通过继承实现。
 6        MethodInterceptor cglibProxyInterceptor = new CglibProxyInterceptor();
 7        Enhancer enhancer = new Enhancer();
 8        // 设置父类（被代理的类）
 9        enhancer.setSuperclass(service.getClass());
10        // 设置实现了MethodInterceptor接口的interceptor对象
11        enhancer.setCallback(cglibProxyInterceptor);
12        // 创建代理类实例。（实际是创建了一个被代理类的子类）
13        Service serviceDynamicProxy_2 = (Service) enhancer.create();
14        serviceDynamicProxy_2.func_2();
15    }
16}
17
18// 输出
19// pre of CGLIB proxy
20// ServiceImpl.func_2
21// post of CGLIB proxy

 1import java.lang.reflect.Method;
 2
 3import net.sf.cglib.proxy.CallbackFilter;
 4
 5public class CglibProxyInterceptorFilter implements CallbackFilter {
 6    @Override
 7    public int accept(Method method) {
 8        if (method.getName().equals("func_1")) {
 9            return 0; // 使用第1个代理类拦截方法
10        }else {
11            return 1; // 使用第2个代理类拦截方法
12        }
13    }    
14}
15
16
17public class ProxyMain {
18    public static void main(String[] args) {
19        Service service = new ServiceImpl();
20
21        // CGLIB 动态代理，通过继承实现。
22        MethodInterceptor cglibProxyInterceptor = new CglibProxyInterceptor();
23        Enhancer enhancer = new Enhancer();
24        // 设置父类（被代理的类）
25        enhancer.setSuperclass(service.getClass());
26        // 设置多个拦截器
27        enhancer.setCallbacks(Callback[]{cglibProxyInterceptor_1, cglibProxyInterceptor_2, NoOp.INSTANCE});
28        enhancer.setCallbackFilter(new CglibProxyInterceptorFilter());
29
30        // 创建代理类实例。（实际是创建了一个被代理类的子类）
31        Service serviceDynamicProxy_2 = (Service) enhancer.create();
32        serviceDynamicProxy_2.func_2();
33    }
34}

基于ASM机制实现，创建业务类的子类作为代理类，在子类中采用方法拦截的技术拦截所有父类方法的调用，顺势织入横切逻辑，来实现动态代理。
