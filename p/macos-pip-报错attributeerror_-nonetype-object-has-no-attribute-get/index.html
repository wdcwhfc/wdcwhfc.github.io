macOS pip 报错：AttributeError_ 'NoneType' object has no attribute 'get'

Mar 17, 2025
1 minute read

0x00

使用pip命令安装依赖库时，如果遇到以下报错，提供以下一种可能的原因及解决方案：

1
2
3

File "<your_python_path>/lib/python3.9/site-packages/pip/_vendor/urllib3/connection.py", line 462, in connect
    if not cert.get("subjectAltName", ()):
AttributeError: 'NoneType' object has no attribute 'get'

0x01

由该原因导致的前提是——你所使用的Python为手动编译或在macOS系统上使用pyenv工具安装。

以下以第二种情况为示例进行说明。

使用pyenv安装python时，需要依赖包括openssl在内的本地库对下载下来的源码进行编译，而macOS使用HomeBrew进行包管理时，本地会有复数版本的openssl库。从上面图片可以看到，pyenv安装指定的python版本时，使用的是openssl 1.1，这就是原因——使用的openssl库版本太老。手动编译同理。

0x02

解决方案1：指定openssl版本

1

PYTHON_CONFIGURE_OPTS="--with-openssl=/opt/Homebrew/Cellar/openssl@3/3.1.2/" pyenv install 3.9.9

可以使用`brew info openssl`命令获取openssl的路径和版本。

解决方案2：临时禁用 SSL 验证

1

pip install 包名 --trusted-host pypi.org --trusted-host files.pythonhosted.org

解决方案3：下载whl文件本地安装

在 https://pypi.org/ 搜索并下载所需的依赖库，本地安装。

如：

1

pip install certifi-2025.1.31-py3-none-any.whl
